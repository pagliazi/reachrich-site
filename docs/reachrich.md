# ReachRich — 总览

> ReachRich 是整套体系的**数据底座**，在调用关系中是**被调用方**：为量化研究提供统一、干净、可靠的多市场数据。上层 RR-Agent（独立项目）通过 API / MCP 向它取数。

## 定位

把"找数据、清数据、对数据"这件最耗人的事做成一层稳定服务。研究者拿到的是**口径一致、复权对齐、多源校验过**的数据，而不是一堆需要自己拼接清洗的原始源。一句话：**让研究者只管用数据，不必再为数据本身耗工。**

## 设计原则

- **单一口径**：同一字段无论来自哪个源，对外都是同一种定义、单位与复权方式。
- **同源同口径**：实时与历史出自同一套口径，回测与盘中可直接衔接。
- **多源不轻信**：任何单源都可能错，交叉校验后再对外。
- **质量可观测**：新鲜度、完整性、合理性持续监控，异常即告警。
- **不泄底**：对外只暴露稳定的数据契约，不暴露来源、主机与实现。

## 解决的问题

- **数据源零散、口径不一** —— 不同来源字段、复权、时区、停牌处理各异，自己拼接极易出错。统一口径、统一复权。
- **实时与历史割裂** —— 盘中实时与历史回测常来自两套数据，口径对不上。这里同源同口径。
- **单源不可靠** —— 任一源都会偶发缺失/延迟/错值。多源交叉校验 + 自动降级。
- **质量无人监控** —— 数据是否新鲜、是否异常、是否漏采需持续盯。内建新鲜度与质量监控。

## 能力一览

| 能力 | 说明 |
|---|---|
| 多市场 | A 股 / 港股 / 美股，统一接入 |
| 多类型 | 行情 · 资讯 · 基本面 · 资金 · 板块（详见[数据覆盖](data-coverage.md)） |
| 复权一致 | 前/后复权 + 除权除息一致处理 |
| 实时+历史 | 同源同口径，无缝衔接 |
| 数据质量 | 多源交叉校验 · 新鲜度监控 · 物理约束校验 |
| 高可用 | 7×24 · 自愈 · 多源容错 |
| 接入 | API / MCP，统一数据契约 |

## 与 RR-Agent 的关系

```
RR-Agent（调用方） ──① 请求数据──▶ ReachRich（被调用方）
                   ◀─② 返回行情 / 资讯 / 因子原料──
```

ReachRich 只负责"把数据做对、做稳、做全"；如何用这些数据挖因子、选股、做组合与执行，由 RR-Agent（量化研究智能体，独立项目）完成。
